# Provisional Mastery States

The repository does not yet approve final mastery labels or numeric cutoffs.
For the first app loop, use conservative internal-only states:

- `insufficient_evidence`: not enough skill-linked evidence for a judgment
- `needs_review`: evidence exists, but automated confidence should stay low
- `developing`: documented error patterns suggest the learner still needs support
- `evidence_positive_but_unapproved`: positive evidence exists, but it is not an approved pass state

These are app-facing provisional states, not final curriculum policy.
